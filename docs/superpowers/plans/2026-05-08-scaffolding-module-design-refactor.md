# Scaffolding / Module-Design 职责重组实施计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 消除 `devforge-project-scaffolding` 与 `devforge-module-design` 之间的时序悖论，通过职责重组让每个阶段的职责单一且可兑现。

**Architecture:** 将 scaffolding 精简为纯基础设施生成器（目录、CI/CD、部署、依赖、测试框架），将代码骨架生成移至 module-design（在 component-spec.xml 生成后立即执行）。同步更新流程文档（devforge-design.md、CLAUDE.md、README.md）保持一致性。

**Tech Stack:** Markdown skill 文件（无代码编译，验证通过 `scripts/package-plugin.py --mode all`）

     - Add module header comment to every generated file:
       ```python
       # Module: {module_id}
       # Component: {component_id}
       # Priority: {P0/P1/P2}
       # Source: component-spec.xml::{component_id}
       # Status: {placeholder/minimal-implemented}
       ```
